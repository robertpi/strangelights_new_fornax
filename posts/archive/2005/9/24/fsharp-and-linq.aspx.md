<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB">I could go on. So why should an F# programmer be interested in this library, if they already have all this functionality available in the MLLib.dll library? There are a couple of reasons; firstly the System.Query adds a few new methods that are missing from the F# list library such as Sum, Min, Max, Average and TypeOf, while each of these functions would be relatively easy to put together in F#, its nice to have them there, tested and ready to go. Perhaps more importantly the is the sear number classes in Framework 2.0 that implement the IEnumberable&lt;T&gt;() interface, nearly every collection implements it. This means just about every collection in the .Net framework is now open to this style of functional programming using System.Query, no need to convert it to a native F# type.<o:p></o:p></SPAN></P>
<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB"><o:p>&nbsp;</o:p></SPAN></P>
<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB">To investigate this style of functional programming I decided to implement the cute example of querying the methods on the System.String object that Anders Hejlsberg demonstrates in <A href="http://channel9.msdn.com/showpost.aspx?postid=114680">this video</A>. For those that haven&#8217;t seen the video the objective is to display the of each instance members of String once along with the number of overloads.<o:p></o:p></SPAN></P>
<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB"><o:p>&nbsp;</o:p></SPAN></P>

We can help make things clearer with a bit of code formatting, but this is still not ideal. We could also improve things by defining at intermediate for all these operations, but then we&#8217;d have to think up 3 new names, which is also not ideal.<o:p></o:p></SPAN></P>

<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB">For me the readability of the code is improved by the lambda definitions being close to the functions they are defined on, however if you look at the sample you notice that now most of the lambdas have been explicitly typed. This because F#&#8217;s type inferences works left to right so it can no longer infer the type of the lambdas from the collections they are to be used with.<o:p></o:p></SPAN></P>
<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB"><o:p>&nbsp;</o:p></SPAN></P>
<P class=MsoNormal style="MARGIN: 0cm 0cm 0pt"><SPAN lang=EN-GB style="mso-ansi-language: EN-GB">Now I think we start to see why the VB.NET and C# teams introduced their new query syntax, it looks like it&#8217;s an aid to help the programmer to stop queries becoming a mass of randomly ordered lambdas.<o:p></o:p></SPAN></P>
